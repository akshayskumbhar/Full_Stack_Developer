CSS Transition

1] Animation: Hover Effect with Smooth Transition

Defination:

CSS Transitions allow properties to change smoothly when a state change occurs (like hover, focus, click).

  1. Animates only when a state change occurs
  2. Needs transition-property and transition-duration at minimum
  3. Works best with properties like transform, color, background-color, opacity.
  4. Triggered by pseudo-classes like :hover, :focus, :active
  5. Lightweight and widely supported across all browsers

Syntax

        selector {
                transition-property: background-color, transform; 
                transition-duration: 0.3;
                transition-timing-function: ease-in-out;
                transition-delay: 0s;
                }
        .box:hover {
                background-color: royalblue; /* Color change on hover */
                transform: scale(1.2); /* Slight zoom */
                }

1.1] transition-property

Defination:

transition-property specifies which CSS properties should be animated when their values change

  1. You must specify at least one property to animate
  2. Works with animatable properties such as color, transform, opacity, width, etc
  3. Accepts multiple properties separated by commas.
  4. Can use the keyword all to animate every animatable change
  5. Avoid animating heavy properties like width or height for performance reasons

Syntax

        button {
        padding: 12px 30px;
        font-size: 18px;
        background-color: #555;
        color: #fff;
        border: none;
        cursor: pointer;
        margin: 50px auto;

        transition: background-color 0.3s ease, transform 0.2s ease; 
      }

      button:hover {
        background-color: #007bff; /* Smooth color change */
        transform: scale(1.25); /* Slight zoom */
      }

1.2] Transition-Duration

Defination:

Specifies how long the transition animation should take to complete

  1. Time is defined in seconds (s) or milliseconds (ms)
  2. Every transition-property must have a matching duration
  3. Longer durations create slower, smoother animations
  4. Short durations are best for snappy UI interactions
  5. Can assign multiple duration values for multiple properties

Syntax

        transition-duration: 0.7;
